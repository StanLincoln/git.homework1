1. Как скопировать на свой компьютер репозиторий из github.com?

	На GitHub открываем нужный нам репозиторий и копируем его код SSH. Открываем терминал на компьютере, переходим в каталог, в который мы хотим скопировать репозиторий и используем команду git clone, указав URL репозитория с GitHub. Например: 
git clone git@github.com:StanLincoln/summary.fe.git

2. Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?

	Инициализируйте новый репозиторий в локальном каталоге: git init
	Добавьте файлы в индекс: git add .
	Сделайте коммит: git commit -m "New commit"
	Создайте пустой репозиторий на GitHub.
	Свяжите локальный репозиторий с удаленным на GitHub: git remote add origin git@github.com:StanLincoln/Test.git
	Измените стандартную ветку master: git branch -M main
	Отправьте изменения на GitHub: git push -u origin main

3. Как получить на локальный компьютер изменения с github.com

	Чтобы получить изменения с GitHub на локальный компьютер, используйте команду: git pull origin master

4. Какую информацию нам дают команды:

	git status: Показывает состояние файлов и папок в гит-репозитории. При этом новые файлы отображаються красным цветом и с пометкой Untracked - неотслеживаемые. После добавление этих файлов через команду git add . , их статус меняктся на new file и они отображаются зеленым цветом. Git status  отображает изменения, которые еще не были зафиксированы (в индексе) и изменения, которые уже были зафиксированы, но еще не отправлены в репозиторий.

	git log: Показывает историю коммитов. Отображает список коммитов, их авторов, даты и сообщения о коммитах.
	
	git remote -v: Показывает URL'ы удаленных репозиториев, связанных с локальным репозиторием. Отображает URL для чтения и записи (fetch и push). 

5. Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?

	Используйте команду git checkout file_name для отмены изменений в файле. Все изменения файла можно откатить до последнего коммита.

6. Как вернуть файл из индекса в рабочий каталог?

	Используйте команду git reset для удаления файла из индекса и возврата его в рабочий каталог. Например:
git reset file_name

7. Как отменить ошибочный commit?

	Нужно  сделать команду git status и убедиться, что у нас в индексе нет никаких файлов (modified), т.е. индексное пространство пустое. Затем git commit --amend -m "New commit". При помощи этой команды можно изменить текст коммита, а также добавить или удалить файлы 
в последнем коммите. Эти все действия можно делать, пока коммит не опубликован. 
	Команда git reset HEAD~1 ( или HEAD^ ) возвращает на один коммит назад, его предыдущею версию без каких либо изменений. Теперь можно исправить файлы и сделать новый коммит. Число после HEAD~3 или количество HEAD^^^ указывает, на сколько коммитов мы возвращаемся.

8. Как исправить коммит отправленный на сервер GitHub?

	Команда git revert HEAD создаст новый коммит, который полностью отменяет предыдущий. Нам открывается vi редактор, если мы не изменяем коммит, то выходим с редактора :q! без сохранения или :wq с сохранением.
	Если вы уже отправили коммит на GitHub, выполните git push --force (опасная команда). Эта команда переписывает историю коммитов на GitHub, если мы например локально откатили несколько своих коммитов git reset HEAD~3.
